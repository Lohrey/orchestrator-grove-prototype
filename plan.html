Prototype v7

Orchestrator Grove

Autonauts-inspired browser automation game: bots run looped programs, but the player only talks to an assistant/orchestrator. v7 adds stable refs plus testable base resources, crude tool crafting, plank/pole processing, and Basic Bot assembly requirements.

Open game README Verification screenshot

What changed from v1

DSL bot programs

Templates are data objects with allowlisted ops and a program-counter interpreter. The UI shows the JSON-like program templates.

Local LLM benchmark

Ollama endpoint/model selector, server/local preset buttons, mock vs local mode, JSON tool-call benchmark, latency/eval-count display, fallback validation.

Performance controls

Idle bots park at depot instead of following; max bot cap defaults to 24; FPS cap defaults to 30; bot-list rendering is throttled/capped.

WebGPU path

Progressive WebGPU probe runs when available and reports renderer state; Canvas 2D remains the compatible fallback.

Live voice input

Mic button streams 16 kHz Float32 PCM to a Sherpa-ONNX online Zipformer WebSocket API and inserts partial/final text at the current cursor.

Clickable world

Bots hover/click with DSL menus. Structures right-click into small menus and can append names like sawbench 2 to the chat cursor.

Building UI

Build is now a left-edge drawer with a B shortcut and category tabs; Storage starts with an Item Palette that type-locks on first inserted item.

Modular source

The old large game.js is now a boot file; implementation lives in src/data, src/chat, src/assistant, src/world, and src/utils.

v5 UI

Fullscreen canvas with 1:1 backing-store resize (no stretch), formatted chat bubbles, Enter opens chat, Stats tab in settings, right-edge Bots drawer, left-edge Build drawer, and assignment toast.

Gemma4 backend

Ollama now uses Paul-compatible local model names such as gemma4:12b as the frontend default; Refresh loads the exact names from the browser machine.

World Object Registry

Bots, structures, loose items, and zones expose stable refs like structure:3, item:7, and zone:1 for the LLM/tool-call schema.

Parameterized templates

Programs now have slots for targetSawbench, sourceSawbench, targetFactory, and zone; the bot menu shows resolved steps.

Drawn/radius zones

Players can draw rectangle zones on the canvas or say “small area around sawbench 2” to scope searches.

v7 resource chain

Loose sticks, stones, tree seeds, crude axes, poles, tool benches, and Basic Bot recipe inputs now move through the same bot-program mechanics.

Current DSL shape

{
  "id": "haul_logs",
  "name": "Haul Logs",
  "slots": {
    "targetSawbench": { "kind": "structure", "type": "sawbench" },
    "zone": { "kind": "zone", "optional": true }
  },
  "steps": [
    { "op": "if_inventory", "type": "log", "goto": 3 },
    { "op": "find_item", "type": "log", "zone": "$zone" },
    { "op": "pick_up", "type": "log" },
    { "op": "deliver_to_sawbench", "type": "log", "target": "$targetSawbench" },
    { "op": "loop" }
  ]
}

Allowed ops include find_nearest_treefind_itemmove_to_targetchop_treepick_updeliver_to_sawbenchprocess_sawbenchprocess_polesdeliver_to_workbenchcraft_workbenchdeliver_to_factoryassemble_botidle_parkingloop.

Current resource test loop

  1. Craft crude axes at the tool bench from loose stick + stone.
  2. Use loose crude axes to chop trees into logs; bot axes last 100 hits and felled trees drop sticks and tree seeds.
  3. Haul logs to a sawbench, process logs into planks, then planks into poles.
  4. Run build_bots so the factory receives 1 log + 3 planks + 1 pole + 1 tree_seed and creates a new idle bot.

Remote / local browser Ollama

The public build defaults to /ollama-proxy. To use your own GPU, Settings → Local AI → Use my local Ollama sets http://127.0.0.1:11434, which points at the browser machine. Start/restart Ollama with CORS for this site:

ollama pull gemma4:12b
OLLAMA_HOST=127.0.0.1:11434 OLLAMA_ORIGINS=https://docs.pau1.cloud ollama serve
